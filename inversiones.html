1. Workflow general

La app está diseñada para un ciclo mensual: planificar → ejecutar → registrar → revisar → cerrar.

🗓️ Durante el mes
  1. Armás un Plan de compras al inicio del mes
  2. Ejecutás cada operación en tu broker
  3. Tildás "Hecho" en el Plan → la app crea la operación y actualiza la posición
  4. Si hacés aportes extra → botón + Aporte en la posición
  5. Si vendés algo → botón − Venta en la posición
📸 Fin de mes
  1. Sacás screenshot de cada broker
  2. Mandás screenshots al asistente y te devuelve JSON con precios
  3. Abrís el wizard 📸 Cerrar mes (3 pasos)
  4. Confirmás → snapshot creado, Revisión actualizada
2. Cartera

Es el estado actual de tus inversiones. Cada fila es una posición (broker + ticker).

Columnas clave
  • Cantidad: unidades que poseés (acciones, bonos, cuotapartes, etc.)
  • Precio actual: cotización del momento, en la moneda del activo (ARS o USD)
  • PPC: Precio Promedio de Compra. Tu costo unitario después de varias compras
  • Valor: Cantidad × Precio (en ARS, convirtiendo USD con el TC de Ajustes)
  • G/P latente: Ganancia/Pérdida no realizada = (Precio − PPC) × Cantidad
Categorías

Cada broker tiene una categoría:

  • Gestión Activa (Balanz, Bull Market, Santander, Crypto): operaciones que decidís activamente
  • Ahorro Automático (Cocos): aportes recurrentes que no requieren decisión activa

En Dashboard y Revisión podés filtrar por categoría para ver el rendimiento por separado.

Botones por posición
+ Aporte · agrega plata o unidades
− Venta · vende parcial o total
Editar · cambia precio, PPC, cantidad, moneda
× · elimina la posición
Acciones masivas
  • 📋 Importar precios: pegás un JSON o texto con tickers + precios y se actualiza todo de una vez
  • 📸 Cerrar mes: abre el wizard de cierre mensual
  • + Posición: agrega un activo nuevo manualmente

3. Operaciones

Histórico de compras y ventas. Cada operación tiene fecha, broker, ticker, cantidad, precio y monto.

Compras vs ventas
• Compras: monto positivo. Se suman a los aportes del mes.
• Ventas: monto negativo, cantidad negativa. Se ven con fondo rojo y pill VENTA. Restan del aporte neto del mes.

El total mostrado al filtrar discrimina: "10 operaciones · Compras: $809.197 · Ventas: −$80.000 · Neto: $729.197"

4. Aportes y ventas

Las posiciones tienen dos modos de actualización, según el tipo de activo:

💰 Modo "importe total"

Para fondos colapsados donde llevás cantidad = 1 y precio = importe total. Típico de Cocos.

Aporte $80k → precio +$80k, ppc +$80k
Venta $50k → precio −$50k, ppc proporcional
🪙 Modo "unidades"

Para acciones, CEDEARs, bonos, crypto. Llevás cantidad real y precio unitario.

Aporte $80k a $5k/unidad → +16 unidades, PPC ponderado
Venta 10 unidades @ $5.5k → cantidad −10, PPC se mantiene
📌 La app elige el modo por default según la posición

Si la cantidad actual es 1 → modo "importe total". Si es ≠ 1 → modo "unidades". Podés cambiarlo manualmente si necesitás.

Ganancia realizada en ventas

Cuando vendés, el sistema calcula automáticamente:

Ganancia realizada = Monto recibido − (Cantidad vendida × PPC)

Se guarda en las notas de la operación (ej: "Venta · Ganancia: $1.491") para que después puedas filtrar o analizar.

5. Plan mensual

Lista de compras anticipadas que tildás cuando las ejecutás. Sirve como checklist al inicio del mes.

Cómo crear un plan
  1. Tab Plan → seleccioná el mes (ej: Junio 2026)
  2. Click + Nuevo item
  3. Completá: broker, ticker, tipo, cantidad, precio estimado
  4. El item queda como "pendiente"
Cómo ejecutar el plan
  1. En tu broker, ejecutás la compra real
  2. Volvés a la app y tildás "Hecho" en el item del plan
  3. La app confirma con un popup que muestra "se creará/actualizará la posición X..."
  4. Al confirmar: se crea la operación y se crea o actualiza la posición en Cartera (PPC ponderado si ya existía)

💡 Tip: si el precio real difiere mucho del planeado, primero editás el item del plan, después tildás "Hecho".

Importar plan masivo (JSON)

Si querés cargar 10 items de una sola vez, usá el botón 📋 Importar plan en la pestaña Plan. Pegás un JSON, ves preview de validación, y aplicás. Flujo recomendado:

  1. Pedile al asistente que te genere el JSON del plan del mes
  2. Click 📋 Importar plan → seleccionás mes destino + modo (agregar o reemplazar)
  3. Pegás el JSON → click 🔍 Analizar → ves preview con totales por broker
  4. Click Importar items → todos los items quedan listos para tildar
Formato del JSON: 
[
  {"broker": "Balanz", "ticker": "ALUA", "tipo": "Acción", "cantidad": 60, "precio": 1029},
  {"broker": "Bull Market", "ticker": "WMT", "tipo": "CEDEAR", "cantidad": 15, "precio": 9480},
  {"broker": "Santander", "ticker": "GD30", "tipo": "Bono", "cantidad": 200, "precio": 940}
]

El monto se calcula automáticamente (cantidad × precio). Tipos válidos: Acción, CEDEAR, Bono, ON, Fondo, Crypto. Brokers tal como aparecen en Ajustes.

6. Revisión mensual

Compara dos momentos: el snapshot del mes anterior contra la cartera actual (o snapshot si ya cerrado). Te dice cuánto ganaste/perdiste y cuánto aportaste.

KPIs principales
  • Valor inicial: total del snapshot anterior
  • Aportes: suma de todas las operaciones del mes (compras − ventas)
  • Valor final: total de la cartera actual o snapshot del mes
  • Ganancia del mes: ver fórmula en la sección 9
Etiquetas por activo
NUEVO aparece este mes (no estaba en el snapshot anterior)
VENDIDO estaba pero ya no está en la cartera
SIN POSICIÓN hay operación pero falta la posición en Cartera
Banners de alerta
  • ⚠️ Snapshot desactualizado: tenés posiciones que no están en el snapshot anterior. Botón 🔄 Sincronizar las agrega.
  • ⚠️ Operaciones sin posición: compraste tickers pero no aparecen como holdings. Botón 🔧 Crear posiciones faltantes.
  • ℹ️ TC del snapshot ≠ TC actual: el TC cambió y genera ganancia "fantasma". Botón ♻️ Recalcular al TC actual.

7. Cierre de mes (Wizard)

Guarda una "foto" del cierre de mes que después sirve como base de comparación para el mes siguiente.

Los 3 pasos
Paso 1 — Mes y TC

Definís qué mes cerrás y el TC USD/ARS al cierre. El TC afecta a todas las posiciones en USD (Crypto, bonos USD, ONs).

Paso 2 — Importar precios

Pegás un JSON con los tickers actualizados. El flujo recomendado: mandás screenshots de tus 5 brokers en el chat y el asistente te devuelve el JSON listo. Click "Analizar" para ver preview de cambios antes de aplicar.

Ver formato del JSON 
[
  {"broker": "Bull Market", "ticker": "AMZN", "precio": 2850},
  {"broker": "Balanz", "ticker": "PAMP", "precio": 5200},
  {"broker": "Santander", "ticker": "GD30", "precio": 940},
  ...
]
Paso 3 — Comparativa pre/post

Antes de confirmar, ves el resumen completo: valor inicial, aportes, valor final, ganancia, desglose por broker. Si algo no cuadra, podés volver atrás y ajustar.

🛡️ Backup automático

Cada vez que confirmás un cierre, se guarda automáticamente el estado anterior. Si te arrepentís, vas a Ajustes → ⟲ Revertir último cierre.

8. Snapshots

Son las "fotos" mensuales que la app usa como base de comparación. Se ven listados en la pestaña Revisión.

Botones de cada snapshot
  • 🔄 Sincronizar: agrega al snapshot las posiciones que están en tu cartera actual pero faltan en él
  • ♻️ Recalcular al TC actual: actualiza solo el TC del snapshot al valor de Ajustes
  • 📌 Rebase a hoy: sobrescribe todo el snapshot con el estado actual de la cartera (útil al inicio para definir "punto cero")
  • ×: elimina el snapshot
📌 ¿Cuándo usar Rebase?

Cuando empezás a usar el sistema y querés tratar el momento actual como "punto cero". El rebase elimina diferencias artificiales y deja la ganancia del próximo mes reflejando solo movimientos reales hacia adelante.

9. Fórmulas clave
GANANCIA DEL MES
Ganancia = Valor Final − (Valor Inicial + Aportes)
"Capital invertido" = lo que tenías al inicio + lo que pusiste durante el mes. La ganancia es cuánto creció ese capital por apreciación de precio.
PORCENTAJE
% = Ganancia ÷ (Valor Inicial + Aportes) × 100
El % se calcula sobre el capital invertido (no sobre el valor inicial solo), para que posiciones nuevas tengan base correcta.
PPC PONDERADO (al comprar más)
PPC = (cant_vieja × PPC_viejo + cant_nueva × precio_nuevo) ÷ cant_total
Cuando hacés un aporte modo "unidades", el PPC se recalcula como promedio ponderado del costo histórico.
GANANCIA REALIZADA (al vender)
Ganancia = Monto recibido − (Cantidad vendida × PPC)
El PPC no cambia al vender — sigue siendo el costo promedio histórico.
VALOR EN PESOS (posición USD)
Valor ARS = Cantidad × Precio USD × TC
Cada snapshot guarda su propio TC. La cartera actual usa el TC de Ajustes.
10. Preguntas frecuentes
¿Por qué la ganancia del mes me da negativa por el mismo monto que aporté?
Significa que el activo no se movió de precio entre snapshots, pero contás un aporte. Pasa porque la posición nueva no estaba en el snapshot anterior. Solución: andá a Revisión → tocá 🔧 Crear posiciones faltantes o 🔄 Sincronizar snapshot.
¿Cómo trato un cambio fuerte de TC del USD?
Si el USD subió (ej: de $1.100 a $1.450) y eso te genera ganancia "fantasma" en pesos por las posiciones USD, tenés dos opciones:
  • Aceptar: el efecto cambiario es ganancia real en pesos para vos. Dejá el TC nuevo y listo.
  • Anular: andá al snapshot anterior → ♻️ Recalcular al TC actual. Esto absorbe el cambio cambiario en la base de comparación.
¿Cómo agrego un broker nuevo?
Andá a Ajustes → Brokers y agregalo. Después podés asignarlo a posiciones, operaciones y al plan.
¿Cómo agrego un ticker que no tengo?
Hay 3 formas:
  • Plan: creás item del plan, tildás "Hecho", y la posición se crea automáticamente
  • Manual: Cartera → + Posición → completás todos los campos
  • Operación directa: Operaciones → + Nueva operación → después usás 🔧 Crear posiciones faltantes en Revisión
Me equivoqué al cerrar el mes, ¿se puede revertir?
Sí. Andá a Ajustes → Zona peligrosa → ⟲ Revertir último cierre de mes. Restaura el estado anterior al último cierre que confirmaste. Solo funciona para el último cierre (no podés revertir varios).
¿Dónde se guardan mis datos?
Todo se guarda en localStorage del navegador. No hay servidor ni nube. Para hacer backup manual, usá el botón ↓ Backup arriba a la derecha — genera un JSON que podés guardar en Drive/Dropbox.
¿Cómo restauro un backup?
Andá a Ajustes → Datos → Restaurar desde backup, pegás el JSON del backup y confirmás. Sobreescribe todo el estado actual.
¿Puedo usar la app en mi celular?
Sí. La app es responsive — funciona bien en mobile. Como los datos están en localStorage, cada dispositivo tiene su propio estado. Si querés sincronizar entre PC y mobile, usá el backup JSON: lo exportás de un lado y lo importás del otro.
¿Por qué Cocos lo llevo con cantidad = 1?
Es una convención simplificada. Cocos muestra los FCIs en cuotapartes con valores raros (cuotapartes con 4 decimales × valores también raros). Para simplificar, los llevamos como "cantidad = 1, precio = importe total". Los aportes se hacen en modo "💰 Suma al importe total" para mantener cantidad = 1.
¿Cómo manejo un dividendo o cupón en pesos/dólares?
Si te pagaron un cupón/dividendo que quedó disponible para reinvertir, lo más limpio es:
  • Si reinvertís en el mismo activo: usá + Aporte con el monto del cupón
  • Si reinvertís en otro activo: cargá una operación normal de compra del activo nuevo
  • Si lo retirás: cargá una operación de venta del activo que pagó el cupón por el monto del cupón
Nota: la app no tiene un módulo específico para dividendos/cupones todavía. Si lo usás mucho, se puede agregar.
¿Qué pasa si me equivoco en un precio del JSON de cierre?
Tres opciones:
  • Si todavía no confirmaste el wizard: tocás "← Anterior" y corregís el JSON
  • Si ya confirmaste pero recién: Ajustes → Revertir último cierre
  • Si pasó tiempo: andá a Cartera → Editar la posición y corregí el precio. El snapshot ya queda guardado con el precio viejo, pero podés re-cerrar el mes encima.
¿La app suma comisiones?
No automáticamente. El "monto" de la operación es lo que vos cargás. Si querés que incluya comisiones, sumalas manualmente al monto. Recomendado: cargá el monto bruto (cantidad × precio) y descontá comisiones en una operación aparte (o usalo como nota).
¿Hay límite de cuántos activos puedo tener?
No técnicamente. La app funciona bien con cientos de posiciones. Lo único que crece linealmente es el JSON de backup. Si llegás a 500+ posiciones podríamos pensar en paginación.
11. Glosario
PPC

Precio Promedio de Compra. Costo unitario promedio ponderado de todas tus compras del mismo activo.

Snapshot

Foto del estado de la cartera al cierre de un mes. Contiene precios, cantidades y TC de ese momento.

TC

Tipo de Cambio USD/ARS. Se usa para convertir posiciones USD a pesos.

Aporte

Dinero nuevo que metés a una posición. Se registra como operación con monto positivo.

Ganancia latente

Ganancia/pérdida no realizada. Lo que ganarías/perderías si vendieras hoy a precio actual.

Ganancia realizada

Ganancia/pérdida concretada al vender. Diferencia entre monto recibido y costo (cantidad × PPC).

Rebase

Sobrescribir un snapshot con el estado actual. Útil para definir un "punto cero" sin acarrear distorsiones.

CEDEAR

Certificado de Depósito Argentino. Representa una fracción de una acción extranjera que cotiza en BYMA.

12. Backup y datos

La app guarda todo en localStorage del navegador. No hay servidor. Si limpiás cache o cambiás de dispositivo, perdés todo — por eso hacer backup es crítico.

Cómo hacer backup
  1. Botón ↓ Backup arriba a la derecha → descarga un JSON
  2. Guardalo en Drive, Dropbox, o donde quieras (recomendado: con fecha en el nombre)
  3. Frecuencia recomendada: al menos al cierre de cada mes, idealmente también después de cambios grandes
Cómo restaurar backup
  1. Andá a Ajustes → Datos → Restaurar desde backup
  2. Pegás el JSON
  3. Confirmás → sobreescribe el estado actual con el backup
⚠️ Acciones peligrosas (Ajustes)
  • ⟲ Revertir último cierre: deshace el último cierre de mes. Reversible si lo hacés temprano.
  • Borrar cartera: elimina solo las posiciones. Mantiene operaciones, snapshots y plan.
  • Borrar todos los datos: deja la app vacía. No hay vuelta atrás si no tenés backup previo.
Migración entre versiones

Cuando te paso una versión nueva (ej: v3.8 → v3.9), la primera vez que abrís la app nueva detecta el localStorage de la versión anterior y migra todo automáticamente. No tenés que hacer nada, pero igual te recomiendo hacer backup antes de migrar.
